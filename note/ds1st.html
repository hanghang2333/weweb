线性数据结构

区别于非线性数据结构，线性数据结构中每一个节点只有一个子节点，如数组(一维)、链表、队列、双队列、栈、线性表、串。常见的非线性结构如二维以及多维数组，表格，树等。这只是一个定义，大概知道即可。

链表

链表是一种线性表。线性表有两种实现方式，一个是数组，一个就是链表。二者各有优劣。
数组特点:根据坐标位置能够在O(1)的时间里找到元素。在非末尾插入元素和删除元素均需要移动整个数组，分摊效率为O(n)。元素在物理存储上也是一一紧挨的，空间利用率高，但是如果没有那么大一块连续存储区域则会迫使操作系统进行碎片整理和垃圾回收。
链表特点:在任何位置插入和删除元素的时间为O(1)。访问一个元素需要从链表头依次遍历过去，分摊时间为O(n).元素在物理存储上可以不连续，充分利用碎片。

链表分为单向链表，双向链表，循环链表等。
python中内置的List类型就是链表实现的，可以直接使用。不过其通过某种高效算法使得也可以通过下标直接访问元素。使得使用起来于数组无二致。

问题1:链表反转:
将形状如1->2->3->4->None的链表反转为4->3->2->1->None.
注意这里的null，因为链表长度有限，每一个节点都有指向下一个节点的指针，那么就导致最终一个节点势必要指向一个特殊点来表明链表到头了，这里使用None作为这个特殊值。并且仍然假定这里已经给定了头节点head。
解答:

def reverse(head):
    if head == None or head.next == None:
        return head # 如果头节点是None或者头节点就已经指向None，则无须反转
    pre = None # 反转后的尾巴节点
    while head:
        tmp = head.next # 存储下head节点的下一个
        head.next = pre #要点是要维持一个前序节点pre，因为每次遍历到一个位置，next可以直接获取，而pre则只能是自己之前就特意留下来
        pre = head
        head = tmp
    return pre # 这里为什么不return head？因为在最后一步，pre指向了1，head指向了None，导致循环结束，所以这里应该返回pre

那么，为什么不使用递归来解决,这个问题不是很适合使用递归，因为最后要返回的值不是给的head，而是尾巴。这样就使得一步一步向前走比一步一步入栈递归要更容易。(递归方便人，循环方便计算机)

双向链表

双向链表每个节点有三个值:value、前序节点和后序节点。
双向链表如何反转?(其实反转后还是一样)，不过假设只知道左边的head节点，那么如何强行反转一下。

class DListNode:
    def __init__(self,val):
        self.val = val
        self.pre = self.next = None
     def reverse(self,head) #类里的函数无论需要与否都有一个self参数
         cur = None
         while head: # 循环退出条件为head == None，此时curt为最后一个节点
             cur = head # 要点就是用局部变量存储下head节点，而后再变换next和pre指针。循环完毕后要保证head已经移动到下一个节点上了。这个和单向链表区别在于这里每个节点的next和pre都可以直接获取，无须再额外保存
             head = curt.next
             curt.next = curt.prev
             curt.pre = head
         return curt

链表反转难度并不大，但略微绕，理解时如果在纸上画画会更好理解。在链表循环中，一定要考虑链表当前节点是否为空，尤其是语句中有node.next操作时。

删除节点是重点:
node1->node2->node3删除node2，则只需要:

node1.next = node1.next.next

在node2和node3之间插入node4，只需要:

node2 = node1.next
temp = node2.next
node2.next = node4
node4.next = temp

问题2:如何判定一个链表里是否有循环，也就是是否有环形回路。
解答:这是一种特殊技巧，当碰到和链表有关的问题但是怎么也想不到方法时。快慢指针。
指的是从给定的head开始，设置两个变量fast和slow。fast每次循环都前进两部，即fast=fast.next.next。而slow则只前进一步。主要用于两类问题，一是找整个链表的中间节点(当fast走到尾巴时slow正好走到中间)。二是判定链表是否有环路(如果有环路，则一快一慢总有一天会碰面)。

def has_circle(head):
    slow = head
    fast = head
    while(slow and fast): #只有slow和fast不是None就一直循环，直到break
        fast = fast.next
        slow = slow.next
        if fast: # 进行next操作时一定要确定当前节点不是None
            fast = fast.next
        if fast == slow:#如果fast和slow相等说明有环路
            break
    if fast and slow and (fast==slow):#加强条件,不但需要相等还要保证fast和slow都不是None
    # 因为上面while循环退出的可能有两种，一个是fast或者slow是None，这种情况下链表里肯定不存在环路
    # 一个是相等。之所以这里要加强，是因为fast其实在循环里不一定前进了两步。举例就是假如链表就只有一个元素，那么一次循环后fast也会等于slow，但显然不应该判定为存在环路
        return True
    else:
        return False
队列

FIFO先进先出。常用于优先级队列和深度优先搜索里。
python中的list也可以当做队列使用。如list.append(1)，list.pop(0).append和pop都是O(1)操作.

LIFO后进先出。常用于配对行为的存储。操作系统中的任务切换也常用栈。假设一个单cpu电脑，一个cpu在一个时刻只能运行一个任务进程。而之所以我们感觉像是多个任务在并行是因为cpu在快速的在不同任务间切换。

二叉树

二叉树是每个节点最多有两个子节点的树。子树有左右之分。二叉树的一些概念:
二叉树第i层最多有Math.pow(2,i-1)个节点(根节点视为第1层)。深度为k的二叉树最多有Math.pow(2,k)-1个节点(根节点视为深度为1).一棵深度为k且有Math.pow(2,k)-1个节点的二叉树称为满二叉树.
二叉树某一个节点的孩子数量称为该节点的度数。
只有最下面的两层结点度小于2，并且最下面一层的结点都集中在该层最左边的若干位置的二叉树称为完全二叉树.

三种遍历方法:前序，中序，后序。已知前+中或者后+中则可以恢复二叉树。其中最为关键的是前序遍历中第⼀个⼀定是根，⽽后序遍历最后⼀个⼀定是根，中序遍历在得知根节点后⼜可进⼀步递归得知左右⼦树的根节点。但是这种⽅法也是有适⽤范围的，即元素值不可重复，否则⽆法完成定位。

示例:

class BinaryTree(object):
    def __init__(self):
        self.result = []
    def preorder(self,root):
        if root:
            self.result.append(root.value)
            self.preorder(root.left)
            self.preorder(root.right)

def buildTree(preorder, inorder):#preorder和inorder类型的list[int],即数组
    if inorder:
        ind = inorder.index(preorder.pop(0))
        root = TreeNode(inorder[ind])
        root.left = self.buildTree(preorder, inorder[0:ind])
        root.right = self.buildTree(preorder, inorder[ind+1:])
        return root
哈夫曼压缩

属于信息论里的信息压缩方法，主要的思想是将信息编码。如’我们是我们祖国的人们’。一共有7个字。使用0/1二进制编码。常理来说需要三位，也就是000.三个位置通过变换实现对7个字实现编码。(如000->我,001->们,010->是，011->祖,100->国,101->的，110->人)这样每个字是三位bit。整句话长度为10，则表示这句话需要30个bit。即可以将这句话表示为:000001010000001011100101110001.解码过程三个为一个字依次对应出来即可。

而哈弗曼的思路是想要尽量减少信息占用的空间。按照这样的编码(11->我，01-们，101->是,100->祖,0011->国，0010->的，000->人)。则整句话编码结果为:110110111011000011001000001.一共27位.

理论上来说,哈弗曼压缩是无损压缩里效率最高的了。现有的压缩软件大都是利用其他额外知识来进行编解码所以压缩率比较大。
哈弗曼的总体思路是:出现频率高的信息用较短的bit位表示，出现频率低的用较长的bit位表示。
哈弗曼编码由于编码长度不一致，解码是否会出现问题。这个问题涉及到信息论里的前缀编码的概念，前缀编码的规则是:任何一个字符的编码都不是另一个字符编码的前缀。满足这个规则，即使长度不一，也不会出现解码错误。哈弗曼编码方式满足前缀编码规则。
